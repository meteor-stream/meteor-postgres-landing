Database Everywhere

We implemented PostgresSQL with Node-Postgres on the back-end and miniSQL with AlaSQL (our version of minimongo) on the front-end. We designed a chaining ORM to wrap both, creating one interface for both to maintain Meteor's 'One Language' principle.

Full Stack Reactivity

Although PostgreSQL does not have oplog capabilities, we were able to use Postgres's Notify functionality. With Notify, we were able to automatically synchronize the server and the clients using DDP.

Latency Compensation

We transact with the local database, minisql, before we invoke the server side transaction. When the client receives the notification from the client of the new transaction, it updates the local database to ensure correct information.
